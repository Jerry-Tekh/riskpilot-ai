# Verifiable Usage Evidence

This folder holds RiskPilot AI's verifiable trading record for the hackathon
submission (Track 1 requires a live/paper trading log with timestamp, pair,
direction, price, quantity, and account balance change).

## `trade-log-sample.csv`

A simulated 90-day trading record produced by RiskPilot's **real decision +
risk engine** (not hand-written). Columns:

| Column | Meaning |
|---|---|
| `timestamp` | ISO-8601 time of the trade event |
| `trading_pair` | e.g. BTCUSDT / ETHUSDT / DOGEUSDT |
| `action` | OPEN or CLOSE |
| `direction` | BUY / SELL |
| `price` | fill price |
| `quantity` | position size |
| `realized_pnl` | realized PnL on CLOSE events |
| `account_balance` | running account balance after the event |

**Headline result:** starting balance 10,000 USDT → final 10,896.87 USDT
(+8.97%) at 3.98% max drawdown over the 90-day window, with autonomous
stop-loss / take-profit closes and 58 risk-vetoed (rejected) decisions logged
separately. Note: the risk engine refused every short signal in this window
(unconfirmed-short guard) — shorting a high-volatility asset carries unbounded
upside risk, so the agent stays flat unless conviction is strong and volatility calm.

## Reproduce it

The record is generated by the same engine the live app uses:

```bash
cd server
npm install
npm run seed          # runs the real engine over 90 days of market data
# then download the live CSV from the running API:
curl http://localhost:4000/api/trades/export.csv -o trade-log.csv
```

The deployed app also exposes a **"Export trade log (CSV)"** button on the
Trade Log page, and the live endpoint is:
`https://riskpilot-ai-production.up.railway.app/api/trades/export.csv`
